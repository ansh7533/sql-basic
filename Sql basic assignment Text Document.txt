### Q9: Primary Key vs Foreign Key

**Primary Key:**
- Uniquely identifies each record
- Cannot be NULL
- One per table
- Example: employee_id

**Foreign Key:**
- Links two tables
- References another table's primary key
- Can be NULL (unless specified)
- Multiple per table
- Maintains referential integrity

### Q10: SQL Constraints

**Definition:** Rules applied to columns to ensure data integrity

**Types:**
1. **PRIMARY KEY** - Unique identifier
2. **FOREIGN KEY** - Links tables
3. **NOT NULL** - Prevents null values
4. **UNIQUE** - Ensures distinct values
5. **CHECK** - Validates conditions
6. **DEFAULT** - Sets default values

**Why Use?**
- Ensures data accuracy
- Prevents invalid data
- Maintains relationships
- Enforces business rules
